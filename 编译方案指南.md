# Android项目编译方案（无Android Studio）

## 方案概览

由于您没有Android Studio，我提供以下编译方案：

## 方案1：Gradle命令行编译（推荐）

### 前提条件
1. 安装JDK 8+
2. 安装Android SDK
3. 安装Gradle

### 步骤

#### 步骤1：安装JDK
```bash
# 检查Java版本
java -version

# 如果没有JDK，下载安装JDK 8或更高版本
```

#### 步骤2：安装Android SDK
1. 下载Android Command Line Tools
2. 解压并设置ANDROID_HOME环境变量
3. 安装必要的SDK组件：
```bash
sdkmanager "platforms;android-34"
sdkmanager "build-tools;34.0.0"
sdkmanager "platform-tools"
```

#### 步骤3：安装Gradle
```bash
# 方法1：使用包管理器安装
chocolatey install gradle

# 方法2：手动下载安装
# 从 https://gradle.org/releases/ 下载并配置PATH
```

#### 步骤4：编译项目
```bash
cd h:\AI临时\momo_auto_reply
gradlew assembleDebug  # Windows
# 或
./gradlew assembleDebug  # Linux/Mac
```

## 方案2：在线编译平台

### GitHub Actions（免费）
1. 将项目上传到GitHub
2. 创建 `.github/workflows/build.yml`：
```yaml
name: Build Android APK
on:
  push:
    branches: [ main ]
jobs:
  build:
    runs-on: ubuntu-latest
    steps:
    - uses: actions/checkout@v3
    - name: Set up JDK 11
      uses: actions/setup-java@v3
      with:
        java-version: '11'
        distribution: 'temurin'
    - name: Setup Android SDK
      uses: android-actions/setup-android@v2
    - name: Build with Gradle
      run: ./gradlew assembleDebug
    - name: Upload APK
      uses: actions/upload-artifact@v3
      with:
        name: app-debug
        path: app/build/outputs/apk/debug/app-debug.apk
```

## 问题排查

### 常见错误及解决方案

1. **"ANDROID_HOME not set"**
   - 安装Android SDK
   - 设置环境变量：ANDROID_HOME=C:\Users\[用户名]\AppData\Local\Android\Sdk

2. **"SDK not found"**
   - 使用sdkmanager安装必要组件
   - 确认SDK路径正确

3. **"Gradle version incompatible"**
   - 更新gradle-wrapper.properties中的版本
   - 或安装匹配的Gradle版本

4. **"Build Tools version not found"**
   - 安装指定版本的build-tools
   - 或更新app/build.gradle中的版本
